# 架构设计

## 总体边界

GoSend 采用单进程、分层模块结构，最终产物为包含 Web 静态资源的单个 Go 二进制。

```text
Browser
   │ GoSend Web API
   ▼
Application services ───── Store interface
   │                          │
   ├── Discovery              ├── memory / SQLite
   └── Transfer               └── MySQL / PostgreSQL
          │
          ▼
 LocalSend v2.1 peers
```

计划中的代码边界：

- `internal/localsend`：LocalSend 协议常量与 DTO，不包含网络和 UI 规则。
- `internal/device`：在线设备状态、超时淘汰和信任关系。
- `internal/discovery`：多网卡 UDP 组播、LocalSend HTTPS 注册端点和指纹校验回连。
- `internal/transfer`：发送/接收会话、多文件状态机、进度、取消与文件落盘。
- `internal/store`：统一仓储接口、内存实现以及 SQLite/MySQL/PostgreSQL 适配器和迁移。
- `internal/webapi`：仅供 GoSend Web 界面使用的 JSON API。
- `internal/app`：依赖装配、后台任务和优雅退出。

## 数据与文件目录

- `data-dir`：SQLite、证书和运行状态等持久化数据。
- `send-dir`：Web 界面只能选择此目录内的普通文件作为发送源。
- `receive-dir`：所有接收文件最终只能写入此目录。

发送目录和接收目录必须不同。文件 API 必须以规范化后的根目录为安全边界，拒绝 `..`、绝对路径、目录穿越和越界符号链接。

接收文件先写入 `receive-dir/.gosend-tmp`，验证大小及可选 SHA-256 后再原子移动到目标名称。服务崩溃后可识别并清理未完成临时文件。

## 并发模型

- 一个发现服务维护带最后在线时间的设备快照。
- 一个传输管理器拥有全部活动会话，其他层通过命令调用它，避免多个状态写入者。
- 一个会话包含多个文件任务；不同文件可受控并发，同一文件保持顺序流式写入。
- 传输不把完整文件读入内存，进度通过事件发布给 Web 层。

## 安全基线

- LocalSend 端口默认同时使用 TCP/UDP `53317`，Web 管理端口默认 `8080`。
- 以 LocalSend v2.1 规范为实现基线；线上设备信息的 `version` 字段按规范使用 `2.0`。
- LocalSend 兼容层实现 HTTPS、自签名证书指纹校验和明确的 HTTP 降级开关。
- 上传令牌绑定会话、文件和对端地址，并使用密码学安全随机数。
- 所有请求设置大小限制、读取/写入超时和并发上限。
- Web 管理面默认仅适用于可信局域网；公网暴露前必须启用反向代理认证或后续内置认证。
- 信任设备不等价于只信任显示名称，身份以证书指纹或持久随机指纹为准。

## 持久化

支持四类数据库后端：

- `memory`：无外部依赖、进程内有效，适合测试或无持久化临时节点。
- `sqlite`：默认后端，纯 Go 驱动，无 CGO，适合单节点、树莓派和 NAS。
- `mysql`：兼容 MySQL/MariaDB 部署。
- `postgres`：兼容 PostgreSQL；配置别名 `pgsql`、`postgresql` 会规范化为 `postgres`。

数据库表包含：

- `settings`
- `trusted_devices`
- `transfer_sessions`
- `transfer_files`

发现到的在线设备首先保存在内存；信任关系和传输历史持久化。三个 SQL 后端共享仓储契约但使用各自的版本化 DDL。SQLite 限制为单连接并启用 WAL、外键和 5 秒忙等待；外部数据库使用连接池。

所有时间以 UTC RFC3339Nano 文本保存，避免跨数据库时区和精度差异。数据库 DSN 只从配置读取，不写入状态接口或日志。

## 节点身份

首次启动在 `data-dir/identity.pem` 生成 ECDSA P-256 自签名证书，私钥文件权限为 `0600`。后续启动复用该证书，并以证书 DER 的 SHA-256 作为 LocalSend HTTPS 指纹。数据库切换不会改变节点身份。

## 设备发现

- 在所有启用且支持组播的非回环 IPv4 网卡上加入 `224.0.0.167:<localsend-port>`。
- 启动时立即公告，此后默认每 30 秒公告一次；组播 TTL 为 1。
- 收到 `announce=true` 时优先回连对端 `/api/localsend/v2/register`，失败则发送 `announce=false` 组播响应。
- HTTPS 回连不使用系统 CA 验证自签名证书，而是严格比较发现消息声明的 SHA-256 指纹。
- 在线设备以指纹去重，默认 90 秒未刷新即淘汰。
- 注册请求和响应限制为 64 KiB，并验证协议版本、端口和协议类型。
- 启动时及每 60 秒扫描活动物理网卡所在 IPv4 子网，通过 `/api/localsend/v2/register` 执行规范中的 HTTP 兜底发现；单个宽网段收敛为当前地址所在 `/24`，并限制并发数和最大主机数。
- Web 可通过 `POST /api/v1/discovery/scan` 立即触发一次兜底扫描；重复请求会复用正在执行的扫描。

## 文件接收

一次 `prepare-upload` 对应一个传输会话和多个文件记录，每个文件获得独立的 192 位随机令牌。令牌、会话、协议文件 ID 和来源 IP 必须同时匹配才能上传。

LocalSend HTTPS 服务只设置请求头和空闲连接超时，不设置覆盖整个请求的全局读写期限。手动审批等待和大文件上传可能持续超过十秒，必须由会话取消、请求上下文、文件大小及存储空间限制负责终止。

接收流程：

1. 校验发送者、文件数量、文件名、单会话总大小和可用磁盘空间。
2. 按 `manual`、`trusted` 或 `auto` 策略决定是否接受。
3. 文件流式写入权限为 `0700` 的 `.gosend-tmp`，读取上限为声明大小加 1 字节。
4. 校验实际大小及可选 SHA-256。
5. 使用同文件系统硬链接原子创建最终文件，确保不会覆盖并发创建的同名目标，再删除临时链接。
6. 保留安全的相对目录结构，同名文件自动追加 ` (n)`；拒绝反斜杠、`.`、`..`、绝对路径、符号链接目录和越界名称。
7. 文件及会话状态同步写入统一仓储；取消会话会中止活动读取并清理临时文件。

## 文件发送

- Web/API 只提交固定发送目录内的相对路径；规范化路径及解析后的符号链接都必须仍位于目录根内。
- 准备阶段流式计算 SHA-256、MIME、大小和修改时间，并持久化一个出站多文件会话。
- 使用发现记录中的 IP、端口和协议建立连接；HTTPS 证书必须与发现指纹完全匹配。
- 一次 `prepare-upload` 获取远端会话和逐文件令牌，随后最多并发上传 3 个普通文件。
- 固定发送目录通过受边界保护的目录浏览 API 展示；选择目录后在服务端递归展开普通文件，并在 LocalSend 文件名中保留相对目录结构。
- 活动任务维护文件级内存进度，完成、失败或取消状态持久化。
- 用户取消会同时取消本地请求并调用远端 `cancel`；部分文件失败不会阻止已经开始的其他文件收尾。

## Web 管理面

Web 界面是不依赖外部 CDN 的嵌入式 HTML/CSS/JavaScript，随 Go 二进制发布。JSON API 负责目录浏览和有副作用的操作；SSE 建立时发送完整快照，之后由设备、接收请求、信任关系和传输状态变化触发推送。20 秒注释心跳只用于维持代理连接，浏览器不再定时轮询业务 API。

发送进度快照包含目标设备公开信息。存在活动发送会话时，Web 隐藏常规导航并渲染发送端到接收端的全屏流向页；准备阶段显示等待响应，上传阶段显示聚合进度，取消操作仍调用统一的发送会话取消 API。

历史快照按传输会话携带文件详情，Web 再按文件平铺展示。删除单条文件历史时，空会话会同步清理；清空和单条删除只影响数据库记录，不删除收发目录中的实体文件。进行中的会话禁止删除。文件内容入口仍受 Web 认证和固定目录边界保护。
